IF IT'S NOT WORKING YET
═════════════════════════════════════════════════════════════════════════════

Check 1: What does the console say?

  If you see: 📡 Subscription status for retailer: CLOSED
    → The database publication is still not set up correctly
    → Re-run Step 4 above (ALTER PUBLICATION command)

  If you see: 📡 Subscription status for retailer: SUBSCRIBED
    → But no toast appears
    → Issue is in Toast component (not real-time)
    → Check that your browser supports WebSockets

  If you see: 🔔 Real-time campaign received
    → But wrong role (doesn't match)
    → Check that campaign has target_role='retailer'

  If you see: ⚠️ Campaign target_role does not match
    → Admin created campaign for 'distributor', not 'retailer'
    → In admin portal, make sure "Target Role" is set to "Retailer"

Check 2: Is the database set up?

  In Supabase SQL Editor, run:
